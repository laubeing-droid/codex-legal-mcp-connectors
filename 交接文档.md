## 一、仓库定位

独立的 MCP 连接器配置仓库，不涉及 skills/ 或法律工作流内容。
职责范围：

1. **配置写入**：向 Codex Desktop / Claude Code / Claude Desktop 的配置文件中写入中国法律 MCP 连接器
2. **全环境适配**：自动检测本机客户端环境，输出对应格式（TOML / JSON）
3. **交互式安装**：提示输入 API Key / Access Token，可选安装北大法宝服务
4. **诊断维护**：验证配置状态、检测凭证过期、检查 npm 包版本
5. **版本监控**：GitHub Actions 每周自动检查 chineselaw-mcp / @pkulaw/mcp-cli 版本

### 不涉及

- SKILL.md 包装层、法律工作流内容 → 由 upstream 或 Claude-for-Legal-CN-to-Codex 管理
- Python 依赖、虚拟环境、数据库

## 三、连接器清单

### 1. chineselaw（元典智库）— 首选

| 项目 | 内容 |
|------|------|
| npm 包 | chineselaw-mcp（作者 zooges，MIT） |
| 注册 | https://open.chineselaw.com → API 管理 → 创建 API Key |
| 类型 | stdio（npx -y chineselaw-mcp） |
| 工具数 | 33（法规5 + 案例4 + 企业24） |
| 前置依赖 | Node.js >= 18 |

Codex 配置段：

`toml
[mcp_servers.chineselaw]
command = "npx"
args = ["-y", "chineselaw-mcp"]
startup_timeout_sec = 30
tool_timeout_sec = 600
enabled = true

[mcp_servers.chineselaw.env]
CHINESELAW_API_KEY = "YOUR_API_KEY"    # ← 替换
`

Claude 配置段（JSON）：

`json
{ "mcpServers": { "chineselaw": { "command": "npx", "args": ["-y", "chineselaw-mcp"], "env": { "CHINESELAW_API_KEY": "YOUR_API_KEY" } } } }
`

## 六、已关闭的已知问题

以下问题在 v1.0.0 -> v1.2.0 迭代中已全部解决：

### 1. Token 占位符检测（v1.1.0 解决）
- **问题**：install 时如未输入凭证，写入 `YOUR_API_KEY` / `YOUR_ACCESS_TOKEN` 占位符
- **解决**：verify.ps1 / update.ps1 均增加占位符检测逻辑，update.ps1 的 [4/5] 步骤专门处理

### 2. Token 过期提醒（v1.1.0 解决）
- **问题**：北大法宝 Access Token 过期时无提示
- **解决**：update.ps1 集成 `@pkulaw/mcp-cli update` 验证 Token 有效性

### 3. 跨平台支持（v1.1.0 解决）
- **问题**：初始版本仅 Windows PowerShell
- **解决**：新增 `install.sh`、`verify.sh`、`update.sh`、`uninstall.sh`、`detect.sh`，全部 Bash 实现

### 4. Node.js 前置检测（v1.1.0 解决）
- **问题**：chineselaw 需要 Node.js >= 18，安装时未检测
- **解决**：install 脚本增加 Test-Command node 检测，未安装时跳过 chineselaw 并给出下载提示

### 5. MCP 配置写死（v1.1.0 解决）
- **问题**：install.ps1 写死 chineselaw + 全部 10 个北大法宝服务
- **解决**：支持选择要安装的北大法宝服务（多选交互）

### 6. 多环境适配（v1.2.0 解决）
- **问题**：只支持 Codex Desktop（TOML 格式）
- **解决**：detect.ps1 / detect.sh 自动识别 Codex Desktop、Claude Code、Claude Desktop，写入对应格式（TOML/JSON）

### 7. update.ps1 MCP 委托（v1.2.0 解决）
- **问题**：`Claude-for-Legal-CN-to-Codex` 的 update.ps1 仍硬编码了 MCP 检查逻辑，未委托给本仓库
- **解决**：本仓库 update.ps1 已成熟，上游 skills 仓库已改为调用本仓库脚本

### 8. 服务中文名（v1.2.0 解决）
- **问题**：北大法宝 10 个服务在安装界面使用英文段名
- **解决**：文档和安装界面均使用北大法宝官方 displayName（中文名），来源为 `@pkulaw/mcp-cli` npm 包内的 `dist/config/servers.json`

### 9. 卸载支持（v1.2.0 解决）
- **问题**：缺少卸载脚本
- **解决**：新增 uninstall.ps1 / uninstall.sh，从 TOML/JSON 配置中移除连接器段

## 十、脚本架构详解

### detect.ps1 / detect.sh — 环境检测模块（核心）

**作用**：被 install.ps1、verify.ps1、update.ps1、uninstall.ps1 通过 `. ./detect.ps1` 加载。

**关键函数**：`Get-EnvironmentInfo`

| 返回值字段 | 说明 |
|-----------|------|
| `Name` | 环境标识：codex / claude-code / claude-desktop |
| `Display` | 显示名：Codex Desktop / Claude Code / Claude Desktop |
| `ConfigPath` | 配置文件的绝对路径 |
| `Format` | 格式：toml / json |
| `Installed` | 布尔值，该客户端是否已安装 |
| `McpSection` | 配置段名：mcp_servers / mcpServers |

**检测逻辑**：
- Codex Desktop：检查 `~/.codex/config.toml` 是否存在
- Claude Code：检查 `~/.claude/settings.json` 是否存在
- Claude Desktop：检查 `%LOCALAPPDATA%/Claude/claude_desktop_config.json`（Win）或 `~/Library/Application Support/Claude/...`（Mac）

如果没有任何客户端存在，install.ps1 会 fallback 为 Codex Desktop（创建 `~/.codex/config.toml`）。

### install.ps1 — 安装脚本（主入口）

| 步骤 | 说明 |
|------|------|
| [1/5] 环境检测 | 调用 detect.ps1 的 Get-EnvironmentInfo，列出所有已安装的 MCP 客户端 |
| [2/5] 前置检查 | 检测 Node.js 是否可用（chineselaw 依赖） |
| [3/5] 输入凭证 | 交互式：输入 chineselaw API Key（可选）、北大法宝 Access Token（可选） |
| [4/5] 选择服务 | 交互式：选择要安装的北大法宝服务（`a`=全部，`1,3,5`=指定序号，留空=跳过） |
| [5/5] 写入配置 | 遍历所有检测到的客户端，写入 TOML 或 JSON 格式配置 |

**关键数据结构**：`` — 北大法宝服务声明数组
包含 name（段名）、displayName（中文名）、endpoint（URL 路径）、description（用途说明）。

### update.ps1 — 更新诊断脚本

|------|------|

| [2/5] npm 版本检查 | 检查 chineselaw-mcp / @pkulaw/mcp-cli 是否有新版 |

### 跨平台差异注意事项

| 项 | Windows (PowerShell) | macOS/Linux (Bash) |
|----|---------------------|---------------------|
| 检测脚本 | detect.ps1 | detect.sh |
| 写入配置 | `Out-File -Encoding UTF8` | `cat > file` |
| JSON 解析 | `ConvertFrom-Json` | `jq` |

## 十一、常见维护问答

### Q: chineselaw-mcp 更新了，需要做什么？
A: GitHub Actions 会创建 Issue。一般无需操作，除非新版有破坏性变更（如环境变量名变化、工具名变化）。
查看 https://github.com/zooges/chineselaw-mcp 的 Release Notes。

### Q: @pkulaw/mcp-cli 更新了，需要做什么？
A: GitHub Actions 会创建 Issue。主要关注两点：
1. 北大法宝 MCP 服务列表是否变化（`dist/config/servers.json`）
2. CLI 命令参数是否变化

### Q: 需要支持新的 MCP 客户端怎么办？
A: 1) 在 detect.ps1 / detect.sh 的 Get-EnvironmentInfo 中新增检测逻辑
2) 确认该客户端的配置路径和格式（TOML/JSON）
3) 在 install.ps1 / verify.ps1 / update.ps1 / uninstall.ps1 中确认格式适配
4) 更新 docs 和交接文档

### Q: 为什么有些代码在 PS1 和 SH 中各有一份？
A: Windows 用户默认无 Bash/Python 环境，PowerShell 是唯一可靠的选择。macOS/Linux 用户默认有 Bash。
两种脚本保持功能对等，修改时必须同步更新双版本。

### Q: 配置文件被用户手动修改了怎么办？
A: install.ps1 采用新增不覆盖策略 — 如果 [mcp_servers.*] 段已存在，跳过写入。
update.ps1 仅检查不写入。uninstall.ps1 会移除对应段。
推荐用户手动修改配置文件后运行 `.\verify.ps1` 验证。

### Q: 如何测试安装脚本？
A: 在测试环境运行全流程：
`.\install.ps1  # 输入测试用 Token`
`.\verify.ps1   # 确认连接器正确写入`
`.\update.ps1   # 确认自更新和诊断正常`
`.\uninstall.ps1  # 确认清理干净`
推荐在 Windows + macOS/Linux 双平台测试。
