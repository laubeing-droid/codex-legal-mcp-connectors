## 一、仓库定位

本仓库是独立的 MCP 连接器管理中心，不依赖任何上游仓库。唯一职责：

> 向本机所有 MCP 客户端配置文件写入 / 验证 / 更新 / 移除中国法律检索连接器。

被 [Claude-for-Legal-CN-to-Codex](https://github.com/laubeing-droid/Claude-for-Legal-CN-to-Codex) 通过 `install.ps1` / `update.ps1` 委托调用。

## 三、连接器配置清单

### 3.1 元典智库（chineselaw）

**方式 A：HTTP MCP（推荐）**
- 3 个分类 Server + 1 个全能力入口
- 端点格式：`https://open.chineselaw.com/mcp/{分类}/stream`
- 鉴权：`Authorization: Bearer YOUR_API_KEY`
- API 文档：https://open.chineselaw.com/docs

**方式 B：npm stdio（备选）**
- `npx chineselaw-mcp`
- 环境变量：`CHINESELAW_API_KEY`

**方式 C：REST API**
- 36 个 REST 端点
- 基础 URL：`https://open.chineselaw.com`
- 鉴权：`X-API-Key: YOUR_API_KEY`

### 3.2 北大法宝（pkulaw）

- 10 个 HTTP MCP 服务
- 基础 URL：`https://apim-gateway.pkulaw.com/mcp/{service}`
- 鉴权：`Authorization: Bearer YOUR_ACCESS_TOKEN`
- CLI 工具：`@pkulaw/mcp-cli`
- 管理后台：https://mcp.pkulaw.com/console/apps

### 3.3 飞书（LarkSuite）

- `npx @larksuiteoapi/lark-mcp`
- 环境变量：`LARK_APP_ID` + `LARK_APP_SECRET`

### 3.4 自建 MCP Server

| Server | 端口 | 鉴权 | 启动方式 |
|--------|------|------|---------|
| flk-npc（法规库） | 18062 | 无（免费） | `python servers/flk-npc/scripts/server.py` |
| rmfyalk（案例库） | 18061 | Cookie Token | `python servers/rmfyalk/scripts/server.py` |

## 四、维护场景

### 场景 A：chineselaw-mcp 更新

GitHub Actions (`npm-monitor.yml`) 会自动检查新版并创建 Issue。维护步骤：
1. 查看 https://github.com/zooges/chineselaw-mcp/releases 确认变更
2. 如有破坏性变更（环境变量名/工具名变化），更新 `install.ps1/sh`
3. 同步更新 `docs/connectors.md`
4. 关闭 Issue

### 场景 B：北大法宝服务列表更新

`@pkulaw/mcp-cli` 的 `dist/config/servers.json` 可能随新版变化。维护步骤：
1. 检查新版本 `npm view @pkulaw/mcp-cli dist-tags`
2. 对比 servers.json 与服务声明数组
3. 如有增减，更新 `install.ps1/sh` 和 `docs/connectors.md`
4. 更新 CHANGELOG

### 场景 C：新增 MCP 客户端支持

1. 在 `detect.ps1/sh` 的 `Get-EnvironmentInfo` 中新增检测逻辑
2. 确认配置路径和格式（TOML / JSON）
3. 在 `install.ps1/sh` 的配置写入步骤适配新格式
4. 在 `verify.ps1/sh`、`update.ps1/sh`、`uninstall.ps1/sh` 中同步支持
5. 更新 `README.md` 的支持环境表格
6. 更新本交接文档

### 场景 D：moyupeng 上游更新

`.github/workflows/upstream-moyupeng.yml` 自动同步。手动触发：
```bash
gh workflow run upstream-moyupeng.yml
```

## 五、发布流程

1. 更新 `CHANGELOG.md`
2. 提交并推送所有修改
3. GitHub Actions 自动运行（无需手动触发）
4. 在 [Releases](https://github.com/laubeing-droid/Codex-Claude-legal-cn-mcp-hub/releases) 创建 Release，标注版本号

## 六、跨平台注意事项

| 项 | Windows（PowerShell） | macOS/Linux（Bash） |
|----|----------------------|---------------------|
| 检测脚本 | detect.ps1 | detect.sh |
| 写入 | `Out-File -Encoding UTF8` | `cat > file` |
| JSON | `ConvertFrom-Json` / `ConvertTo-Json` | `jq` |
| TOML | 正则匹配 | 正则匹配 |
| Config 路径 | `%APPDATA%/Claude/` | `~/Library/Application Support/Claude/` |
| 执行权限 | 直接运行 | 需 `chmod +x` |

修改逻辑时必须双平台同步更新。
